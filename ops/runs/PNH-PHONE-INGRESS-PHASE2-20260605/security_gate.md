# Security Gate

## Verdict

`blocked`

## Reason

No phone-reachable Windows private LAN IP is available. Applying firewall or
portproxy rules now would either do nothing useful or risk exposing the
companion through a public interface.

## Required Controls When Resumed

- Use only `safePhoneUrls` generated by `scripts/phone_ingress_reachability_check.py`.
- Keep `--allowed-origin` exact and private-LAN-only.
- Run companion with `--enable-encrypted-vault` and `--vault-passphrase-provider windows-dpapi-file`.
- Use synthetic or low-risk input first.
- Stop the companion after the phone test.

## Explicitly Rejected

- Public IP origin.
- Wildcard CORS.
- `0.0.0.0` as browser URL or allowed origin.
- WSL NAT address as phone URL.
- Phone ingress on untrusted network.
